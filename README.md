# Converse — Distributed Real-Time Messaging Platform

Chatify is a **full-stack real-time messaging platform** built with **Spring Boot (backend)** and **React + Vite (frontend)**.

The system demonstrates real-world messaging architecture including:

* real-time messaging using **WebSockets (STOMP)**
* **JWT-based authentication**
* **direct and group chats**
* **paginated message history**
* **presence tracking**
* **containerized infrastructure** with **Postgres, Redis, and Kafka**

The entire system can be run locally using **Docker Compose**, providing a production-like development environment.

---

# Key Features

## Core Messaging

* JWT authentication (access + refresh token flow)
* Direct messaging (1-to-1)
* Group chat support
* Real-time messaging via WebSocket/STOMP
* Paginated message history loading
* Message delivery / read tracking
* Presence indicators (online / offline)

---

## File Attachments

* File upload support for chat messages via **AWS S3**
* Supported formats:
  * images (JPEG, PNG, GIF, WEBP)
  * PDF
  * DOC / DOCX
  * Video (MP4, QuickTime, AVI)

Endpoints:
```
POST /api/files/presigned-url
PUT {presigned-url} (Direct to S3)
```

# Option A — Run with Docker (Recommended)

Docker stack includes:
* Postgres
* Redis
* Kafka
* Zookeeper
* Spring Boot backend
* React frontend served via Nginx

---

## 1. Create `.env`

In the project root:

```env
POSTGRES_DB=chatify
POSTGRES_USER=chatuser
POSTGRES_PASSWORD=change-me

REDIS_PASSWORD=change-me

JWT_SECRET=change-me-secret

GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_S3_BUCKET_NAME=
AWS_S3_REGION=
```

---

## 2. Start the full stack

```bash
docker compose up --build
```

---

## 3. Access the application

Frontend:
```
http://localhost
```

Backend API:
```
http://localhost:8080
```

WebSocket endpoint:
```
http://localhost:8080/ws
```

# Troubleshooting

## WebSocket connection fails

Check:
* backend running on port 8080
* correct WebSocket endpoint
* CORS configuration

---

## Database connection issues

Ensure Postgres container is running and:
```env
SPRING_DATASOURCE_URL=jdbc:postgresql://postgres:5432/chatify
```

---

## Frontend build errors

Ensure frontend dependencies are browser-compatible and avoid Node-only modules in client code.
